poxicle Documentation

Design and technical reference for poxicle — a toolkit- and desktop-agnostic edge-particle engine for Linux, with its own GPU renderer. Last updated 2026-06-29.

poxicle draws fast, high-quality animated effects around the edges of a surface — bursts, tails, pulses, beams, ambient shimmer. The reusable core is pure C with zero dependencies: it simulates a particle stream and emits a flat PoxInstance[], which a GLES3 renderer draws in one instanced call with shapes as signed-distance fields. Everything environment-specific — how a surface is created and hosted — is an opt-in backend you can ignore, so the same engine drives a Wayland subsurface overlay, a KWin Plasma 6 compositor effect, a GNOME Shell extension, and a terminal. These pages document each part from its real implementation.

Architecture & Core

  • The Simulation Core CORE
    The pure-C engine: perimeter math, tunables and envelopes, bursts, the geometric kinds, the tick loop, and seeded clocks.
  • The GLES Renderer CORE
    One glDrawArraysInstanced per frame, shapes as signed-distance fields, the vertex/fragment shaders, and premultiplied blending.
  • The poxbridge Protocol CORE
